# v0.5 开发笔记

## 1. OpenClaw Agent WS 连通

**思路**: OpenClaw gateway 在 `ws://localhost:18789` 上运行，OPC adapter 通过 WebSocket 连接并执行 RPC 协议。

**踩坑**:
- **死锁**: `Start()` 持有 `mu.Lock()` 不释放，`readLoop()` 要 `mu.RLock()` 才能读消息 → 互相等待。拆分锁粒度解决。
- **Challenge 协议**: Gateway 连接后先发 `connect.challenge` event（带 nonce），adapter 原来直接发 `connect` 忽略了这一步。readLoop 把 event 全部丢弃。
- **client.id 白名单**: OpenClaw 校验 `client.id` 必须是 12 个预定义常量之一（`webchat-ui`, `gateway-client` 等），不能随便填。翻了 OpenClaw 源码 `src/gateway/protocol/client-info.ts` 才找到。
- **Device Identity**: Gateway 要求 ED25519 签名认证。生成密钥对 → sha256 fingerprint → 签名 `v3|deviceId|clientId|...` payload。
- **Gateway Token**: 签名通过后还要 auth token，从 `~/.openclaw/openclaw.json` 的 `gateway.auth.token` 字段读取，通过 `OPENCLAW_GATEWAY_TOKEN` 环境变量传入。

## 2. Goal AI 智能自动分解

**思路**: 用户只创建 Goal，系统调用 AI（通过 claude-code agent）自动拆解为 Project → Task → Issue，走 Plan → Approve → Execute 三段式。

**实现**:
- `Decomposer` 接口化，原实现改名 `StaticDecomposer`，新增 `AIDecomposer`
- AI 通过 `opc-decomposer` 系统 agent 调用 claude，prompt 要求返回严格 JSON
- `extractJSON()` 处理 AI 回复中的 markdown code fence 包裹
- 失败最多重试 2 次
- Guardrails: maxProjects / maxTasksPerProject / maxAgents 约束校验

**踩坑**:
- AI agent 的 types 定义放在 `goal` 包但 `ai_decomposer.go` 引用了 `v1.AIDecomposeResult`，合并时类型不匹配。
- `MaxCostDollars` vs `MaxBudget`（string vs int）命名不一致，prompt.go 编译失败。
- `controller.ExecuteTask` 返回 `adapter.ExecuteResult` 但 goal 包不能直接导入 adapter（循环依赖），用 `AgentController` 接口 + `ExecuteResult` 镜像类型解决。

## 4. Federation 完整链路

**思路**: 主 OPC 注册联邦公司 → 发送 Goal 到远程 OPC → 远程执行 → 回调通知 → 主 OPC 标记完成。

**实现**:
- 后端: `POST /api/goals/federated`（创建并分发）、`POST /api/federation/callback`（接收回调）
- 回调处理: 更新本地 Goal/Project/Issue 状态和 token/cost，所有 project 完成 → Goal 标记 completed
- 远程 `runTask` 完成后异步 POST callback
- 前端: 公司卡片展开可查看远程 agents/tasks/metrics，"打开公司面板"跳转远程 dashboard，"Create Goal" modal 选择目标公司

**踩坑**: 无特别问题（Federation agent 独立实现，代码较长但逻辑清晰）。

**踩坑（合并时）**:
- 3 个 worktree agent 并行修改 server.go，Federation agent 覆盖了整个文件导致 KindGoal 和 Company case 丢失。手动恢复后确保 switch 包含所有 Kind。
- `RunTaskSync` 方法被 AI agent 的 client.go 覆盖删除，run.go 编译失败。改为 `RunTask`。
- Store interface 缺少 Project/Issue/DeleteGoal/Stats 方法，需要同时在 sqlite 和 postgres 实现。
- workflow.steps 可能为 undefined（API 返回的 WorkflowRecord 只有 specYaml 没有 parsed steps），前端 `.map()` 崩溃，加 `|| []` 防护。
